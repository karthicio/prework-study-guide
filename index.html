HTML

  • HTML focuses on on structuring pages.
  • HTML elements label pieces of content such as "this is a heading", "this is a paragraph", "this is a link", etc.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The body element can be further broken up into sections to distinguish parts of the page.

CSS

  • CSS code determines how HTML elements should be displayed.
  • CSS is used to control different parts of individual parameters such as color, shape, size and the overall appearance of webpage parts
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • All git use must begin with a repository on some central server that the user connects to
  • git: must precede every Git statement
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull: pulls a branch's changes into the current branch (depends on branch)
  • git push: push the changes we have locally to our remote GitHub branch, can create a new remote branch or add new changes to an existing branch.
  • git checkout: to move the working branch to a new branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Functions are pieces of code that can be called and used repeatedly after they are made.
  • Loops are repeating pieces of code that can be made to repeat any numbers of times
  • Conditional statments make a differing descision in code based on whether or not a given parameter is true (If X then Y else Z)
